## Architecture Flow

### Smart Caching Flow
```
User/Agent Request
    ↓
Agent Tool (get_wisdom_packet)
    ↓
HttpBimbaClient wrapper
    ↓
BimbaGraphQLClient (GraphQL query)
    ↓
Backend GraphQL Endpoint (/graphql)
    ↓
Ariadne Resolver (resolve_get_wisdom_packet)
    ↓
WisdomPacketService
    ↓
├─ If force_regenerate=True → Invalidate cache
├─ Check Redis cache (24h TTL)
├─ If cache HIT → Return cached packet
└─ If cache MISS:
    ├─ Neo4j: traverse_subgraph(coordinate, depth)
    ├─ Extract key concepts via pattern recognition
    ├─ Generate narrative synthesis
    ├─ Detect apophatic pointers (missing themes)
    ├─ Calculate synthesis quality score
    ├─ Cache result in Redis (24h TTL)
    └─ Return WisdomPacket
```

## Performance Characteristics

### Cache Behavior
- **First request:** Cache MISS → Full generation → Store in Redis (24h TTL)
- **Subsequent requests:** Cache HIT → Instant retrieval from Redis
- **Force regenerate:** Invalidates cache → Fresh generation → New cache entry

### Synthesis Performance
- **Shallow depth (1-2):** ~100-300ms generation time
- **Medium depth (3-4):** ~300-700ms generation time
- **Deep depth (5):** ~700ms-1.5s generation time
- **Cached retrieval:** ~10-50ms (Redis lookup)

### Synthesis Quality Tuning
The synthesis score calculation in `_calculate_synthesis_score()` uses weighted dimensions:
- **Concept richness** (30%): Number and quality of extracted concepts
- **Narrative quality** (30%): Length and synthesis indicators
- **Subgraph connectivity** (20%): Number of related nodes
- **Apophatic completeness** (20%): Inverse of gaps detected

Scores > 0.7 indicate "genuine contextual insight" per AC 5.
