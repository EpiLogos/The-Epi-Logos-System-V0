# Story 08.08: epii-etymological-atelier-interface

## Status
Ready

## Story
**As a** user seeking philosophical depth through etymological discovery,
**I need** an Epii subsystem page (#5) with modal-expanded access to both Archive (#5-0) and Atelier (#5-5) interfaces that enable deep reading of etymological explorations and active participation in new discoveries,
**so that** I can access the dual modalities of etymological archaeology within the proper coordinate structure, follow linguistic scent-trails, participate in co-creative etymology building with AI partnership, visualize QL-structured word communities, discover resonances with canonical knowledge, and contribute discoveries to the growing tradition.

   - Same layout, animation logic, and transition patterns (not recreated - copied)
   - Epii-specific content (text replacement for Paramasiva philosophical content)
   - PNG swap: `paramasiva-icon.png` → `epii-icon.png`
   - Coordinate context: `#5` instead of `#1`
2. **Modal Expanded State - 2 Cards**: The modal expanded state must show simplified carousel with exactly 2 cards (no scroll arrows/navigation buttons):
   - Card 1: #5-5 Etymological Archaeology (navigates to `/epii/atelier`)
   - Card 2: #5-0 Archive/Document Hub (navigates to `/epii/archive`)
3. **Page Routing Structure**: The system must create proper Next.js routing:
   - `/epii` - Base Epii page (modal collapsed/expanded states)
   - `/epii/archive` - #5-0 Archive interface page
   - `/epii/atelier` - #5-5 Atelier interface page

### #5-0 Archive Interface (Reading Modality) - SEE STORY 08.02
4. **Archive Repository Interface**: The `/epii/archive` page implements **Story 08.02: Document Viewer & Editor Interface**.
   - Sidebar: Recursive coordinate tree showing documents from MongoDB (`bimba_documents`, `pratibimba_documents`)
   - Main Canvas: Document viewing/editing with role-based access (read-only for users, editable for admins)
   - Upload: Admin-only document upload with coordinate assignment (triggers Story 08.03 preprocessing)
   - Analysis: Integration with Story 08.09 deep analysis pipeline
   - **Implementation Details**: See Story 08.02 for complete acceptance criteria and technical specifications

### #5-5 Atelier Interface (Creative Modality)
8. **Atelier Exploration Interface**: The system must provide an active exploration interface where users initiate etymological investigations by entering words, hunches, or questions.
9. **Real-Time Etymology Visualization**: As exploration progresses, the interface must show branching etymological trees (PIE roots → cognates → modern words) with visual QL pattern emergence (3-fold, 4-fold, 6-fold, 12-fold).
10. **Community Building Interaction**: Users must see QL communities crystallize in real-time as they explore, with visual geometric patterns (toroidal 4+2 structure visible when 6-fold completes).
11. **Bimba Resonance Overlay**: The interface must show when discovered etymological patterns resonate with Bimba canonical coordinates, lighting up relevant regions and displaying connection types.
12. **Insight Capture & Integration**: Users must be able to capture synthesis moments ("aha!") with options to: save to personal exploration, propose for archive, or suggest Bimba enrichment (admin-gated).

### Integration & User Experience
13. **Seamless Archive ↔ Atelier Flow**: Users must transition smoothly between reading mode (#5-0) and creation mode (#5-5) - archives provoke exploration, explorations sediment into archives.
14. **Personal Exploration Management**: Users must have dedicated spaces to track their etymological journeys, view history, manage communities they've built, and see insights they've generated.
15. **Visual Network Growth**: The interface must display the user's exploration as a growing graph - nodes (words) connected by edges (etymological relations), densifying over time with color/size indicating depth.
16. **Collaborative Synthesis Visualization**: When insights emerge from AI-human partnership, the interface must clearly show the con-scire (knowing-with) contribution pattern.
17. **Submission & Review Workflow**: Users proposing Bimba enrichments must see their submission status, admin feedback, and approval/rejection reasoning with full transparency.

### Key UX Principles

**Principle 1: Follow Your Nose**
Navigation should feel like scent-following, not menu-clicking. Every word is potentially a portal. Curiosity drives movement, not predetermined structure.

**Principle 2: See Thinking Happen**
In Archive: witness how insights emerged through dialogue
In Atelier: watch networks crystallize as you explore
Both modes make the constitutional activity visible, not hidden.

**Principle 3: Partnership, Not Automation**
The interface must make clear: human intuition + AI support = discovery
Not: AI generates etymology automatically
The con-scire partnership is architecturally necessary.

**Principle 4: Provisional Synthesis Respected**
Quilting moments are adequate without claiming totality. The interface celebrates completion while keeping doors open. "Yes, THAT" doesn't mean "only that forever."

**Principle 5: The Void Remains Unseen**
The 4+2 structure includes implicate depths that can't be directly grasped. The interface should honor this - not everything needs to be explicit. Mystery preserved.

### QL Pattern Geometric Rendering

**3-Fold (Triangle)**:
- Three nodes in triangle formation
- Center shows root/source
- Edges labeled with relationship type

**4-Fold (Square/Cross)**:
- Four nodes circling center
- Center is implicate void (shown as subtle emptiness)
- Forms cross or square depending on view

**6-Fold (Toroidal 4+2)**:
- Four nodes visible in circulation
- Two implicate positions (0 primordial, 5 quintessential)
- When complete, torus visualization appears
- Möbius twist indicated at closure

**12-Fold (Double Helix)**:
- Ascent path (0→5) and descent path (5→0) intertwine
- Can visualize as helix or as two concentric 6-folds
- Shows bi-directional Logos Cycle

### Architectural Learnings

**3-Layer Session System Understanding**:
1. **Thread ID** (AG-UI/frontend): Chat conversation identifier, listed from MongoDB
2. **Session ID** (Redis/backend): OrchestratorSession with metadata support
3. **Context ID** (Google A2A): Agent-to-agent context layer

**Metadata-Based Filtering Pattern**:
- Store coordinate context (`#5-5`) as metadata on thread creation
- Filter thread lists by context tag at query time
- Enables coordinate-specific thread isolation without separate collections
- Reuses existing infrastructure (MongoDB conversations, Redis sessions)

**Key Principle**: Exhaust existing services before creating new ones. The system already had robust session management - we just needed to add filtering capability.
