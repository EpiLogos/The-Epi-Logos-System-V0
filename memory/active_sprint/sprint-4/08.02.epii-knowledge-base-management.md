# Story 08.02: Epii Document Viewer & Editor Interface

## Status
Draft

## Story
**As a** user (admin or read-only),
**I want** a document viewer and editor interface with coordinate-organized document navigation at `/epii/archive`,
**so that** I can view, search, and (if admin) manage documents organized by their Bimba coordinates.

## Dependencies
- **Story 08.03**: Docling Document Preprocessing & LightRAG Ingestion Pipeline (REQUIRED) - Upload triggers preprocessing
- **Story 08.09**: Deep Document Analysis Pipeline (INTEGRATION) - Shows analysis status and Pratibimba documents

## Acceptance Criteria

### Document Viewer & Navigation (ALL USERS)
1. The system must provide a document viewer/editor page at `/epii/archive` with sidebar + canvas layout.
2. The sidebar must display a **recursive coordinate tree** listing all documents from MongoDB collections:
   - `bimba_documents` - Canonical knowledge documents
   - `pratibimba_documents` - Analysis reflection documents (from Story 08.09)
3. Documents must be nested under their assigned Bimba coordinates in tree structure (e.g., #1 → #1-2 → #1-2-3).
4. The coordinate tree must be collapsible/expandable for easy navigation.
5. Clicking a document in the tree must load it in the main viewing canvas.
6. The main canvas must display document content in readable format.

### Document Upload (ADMIN ONLY)
7. Admins must see an upload button/form in the sidebar.
8. The upload form must include:
   - File selection (PDF, DOCX, PPTX, XLSX, HTML supported)
   - **Bimba Coordinate field** (required) - specifies where document is filed
   - Document title (optional - defaults to filename)
9. Uploaded documents must have `bimbaCoordinate` metadata set in MongoDB.
10. Upload must trigger Story 08.03 Docling preprocessing pipeline automatically.

### Document Editing (ADMIN ONLY)
11. Admins must be able to **edit document content** directly in the canvas.
12. Admin edits must save changes back to MongoDB.
13. The interface must display document metadata (coordinate, upload date, file type, processing status).
14. The interface must provide a **Document Metadata Panel** positioned between the document heading and edit controls showing:
    - **Writer Notes**: Contextual notes about document purpose and usage (editable in edit mode)
    - **Development Content**: Development lifecycle context indicating fresh vs stale content (editable in edit mode)
    - **Journal Entry** action button: Placeholder for creating journal entries related to the document
    - **Ask Epii** action button: Placeholder for AI chat interface about the document
15. The metadata panel must be reusable across Archive and Etymology Explorer interfaces.

### Document Analysis Integration (ADMIN ONLY - Story 08.09)
16. The interface must show a "Deep Analysis" button for admin users (triggers Story 08.09 pipeline).
17. The interface must display analysis status from Story 08.09 (preprocessing, analyzing, complete, failed).
18. When analysis completes, the corresponding Pratibimba document must appear in the coordinate tree automatically.

### Access Control
19. Non-admin users must have **read-only access** to all documents (view only, no edit/upload/delete).
20. All create, update, and delete operations must be restricted to admin role.
21. The UI must clearly indicate admin vs read-only mode (show/hide admin buttons based on role).

### MongoDB Collection Specifications

**Collection: `bimba_documents`**
```javascript
{
  _id: ObjectId,
  title: String,
  content: String, // Full document text content
  bimbaCoordinate: String, // e.g., "#1-2-3"
  fileType: String, // "pdf", "docx", "md", etc.
  filePath: String, // Original file path
  uploadedBy: ObjectId, // User ID
  uploadedAt: Date,
  lastModified: Date,
  processingStatus: String, // "pending", "processing", "complete", "failed"
  writer_notes: String, // Contextual notes about document purpose and usage (AC 14)
  development_content: String, // Development lifecycle context - fresh vs stale (AC 14)
  metadata: {
    pageCount: Number,
    wordCount: Number,
    language: String
  }
}
```

**Collection: `pratibimba_documents`** (Created by Story 08.09)
```javascript
{
  _id: ObjectId,
  title: String,
  content: String, // Markdown analysis reflection
  sourceBimbaId: ObjectId, // Links to bimba_documents._id
  sourceBimbaCoordinate: String, 
  analysisDate: Date,
  analysisModel: String, // "claude-3-5-sonnet-20241022"
  batchCount: Number,
  insights: [{ // Structured insights
    concept: String,
    relationships: [String],
    resonances: [String]
  }]
}
```
