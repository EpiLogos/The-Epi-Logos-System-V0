## Story Statement

As a word archaeologist exploring etymological patterns,
I want rich, interactive D3.js force-directed graph visualizations for etymology trees, QL communities, and Bimba resonances,
so that I can visually discover QL patterns, semantic relationships, and canonical resonances through spatial intelligence rather than text-only interfaces.

## Context

Story 08.07 established the Etymology Archaeology workflow with backend implementation and frontend observability (polling hooks, real-time stats, notification banners). The current implementation uses MVP placeholder visualizations (simple lists and cards).

This story replaces those placeholders with production-quality D3.js force-directed graph visualizations for three core panels:

1. **Etymology Tree Panel** - Branching etymological lineages (PIE roots → cognates → modern words)
2. **QL Community Panel** - Geometric crystallization patterns (3/4/6/12-fold structures)
3. **Bimba Resonance Panel** - Coordinate constellation heat maps

These visualizations reveal **QL pattern emergence**, **semantic relationships**, and **canonical resonances** through visual-spatial intelligence.

### D3 Hook Pattern (Foundation)
```typescript
// hooks/useD3.ts
export function useD3<T extends SVGElement>(
  renderFn: (svg: d3.Selection<T, unknown, null, undefined>) => void,
  dependencies: any[]
) {
  const ref = useRef<T>(null);

  useEffect(() => {
    if (ref.current) {
      const svg = d3.select(ref.current);
      renderFn(svg);
    }
  }, dependencies);

  return ref;
}

// hooks/useForceSimulation.ts
export function useForceSimulation(nodes: Node[], edges: Edge[], options: ForceOptions) {
  const [simulationNodes, setSimulationNodes] = useState(nodes);

  useEffect(() => {
    const simulation = d3.forceSimulation(nodes)
      .force('charge', d3.forceManyBody().strength(options.chargeStrength))
      .force('center', d3.forceCenter(options.width / 2, options.height / 2))
      .force('link', d3.forceLink(edges).distance(options.linkDistance))
      .force('collision', d3.forceCollide().radius(options.collisionRadius));

    simulation.on('tick', () => {
      setSimulationNodes([...nodes]); // Trigger re-render
    });

    return () => simulation.stop();
  }, [nodes, edges, options]);

  return simulationNodes;
}
```

### QL Community Geometric Constraints
```typescript
interface GeometricConstraint {
  foldCount: number; // 3, 4, 6, or 12
  vertices: Array<{ x: number; y: number }>; // Pre-calculated geometric positions
  radius: number;
  centerX: number;
  centerY: number;
}

function calculateGeometricPositions(foldCount: number, center: Point, radius: number): Point[] {
  const angleStep = (2 * Math.PI) / foldCount;
  return Array.from({ length: foldCount }, (_, i) => ({
    x: center.x + radius * Math.cos(i * angleStep),
    y: center.y + radius * Math.sin(i * angleStep)
  }));
}

// Apply as D3 force
function geometricConstraintForce(constraint: GeometricConstraint) {
  return (alpha: number) => {
    nodes.forEach((node, i) => {
      const target = constraint.vertices[i % constraint.vertices.length];
      node.vx += (target.x - node.x) * alpha * 0.1;
      node.vy += (target.y - node.y) * alpha * 0.1;
    });
  };
}
```

### Performance Considerations
- **Node limit**: Force simulation performs well up to ~500 nodes. Beyond that:
  - Use Canvas rendering instead of SVG
  - Implement clustering (show aggregate nodes, expand on click)
  - Consider deck.gl for WebGL acceleration (future)
- **Update strategy**: Incremental updates using D3's `data()` join pattern with enter/update/exit
- **Debouncing**: Debounce re-renders when polling data updates (300ms delay)

## Dev Notes

### D3.js Best Practices for React/Next.js
- Use `useRef` + `useEffect` pattern for D3 DOM manipulation
- Avoid mixing React state updates with D3 transitions (choose one)
- For performance: render to Canvas for >500 nodes, SVG for <500
- Use `d3.forceSimulation().stop()` in cleanup to prevent memory leaks

### Force Simulation Tuning
- **chargeStrength**: -30 to -300 (negative = repulsion)
- **linkDistance**: 50-200 (longer = more spread)
- **collisionRadius**: node.radius * 1.5 (prevent overlap)
- **alpha decay**: 0.01-0.05 (lower = longer simulation)

### Geometric Constraint References
- Triangle (3-fold): Equilateral, 120° angles
- Square (4-fold): 90° angles, cross pattern optional
- Hexagon (6-fold): 60° angles, **toroidal 4+2 structure** (4 outer + 2 inner vertices)
- Dodecagon (12-fold): 30° angles, nested hexagons
