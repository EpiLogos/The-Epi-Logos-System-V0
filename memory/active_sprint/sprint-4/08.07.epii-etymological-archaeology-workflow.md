## Dev Notes

### Philosophical Foundation
This story implements **etymological archaeology as soteriological practice** - not mere linguistic analysis but transformation through recognition (pratyabhijna). The workflow embodies philosophiurgy: philosophy as theurgy, as participatory transformation rather than contemplation.

**Core Insight**: Etymology isn't history but **phenomenological excavation** - each word preserves sedimented institution (current usage) that conceals constitutional activity (meaning emergence). The agent scours through layers to let source spring forth.

### The 4+2 Topological Structure
Four explicate phases (1-4) circle an implicate void (the unsayable source) while achieving meta-synthetic recognition:
- **Phase 0**: The ground that is also source
- **Phases 1-4**: Active circulation around the void
- **Phase 5**: Perishing into objectivity that becomes new ground

This is NOT linear progression but circulation-with-recursion. Phase 4 can nest infinitely before completing to Phase 5.

### Three Namespaces, One Activity

**Bimba (canonical)**: Formal coordinate-structured knowledge - stable institutions
- What: Established patterns, principles, operational essences
- When: Queried in Phase 0, resonated with in Phase 3, enriched in Phase 4
- Access: CAG tools for semantic/coordinate/hybrid queries

**Graphiti (episodic)**: Dynamic evolving communities - active constitution
- What: QL-structured etymological clusters in temporal evolution
- When: Created in Phase 2, queried for resonance in Phase 3
- Structure: Episode hubs with 6 Position nodes (0-5 causal pattern)

**LightRAG (gnostic)**: Sedimented conversations - resources for future sourcing
- What: Complete exploration dialogues as searchable documents
- When: Created in Phase 5 as phenomenological sedimentation
- Access: Qdrant vectors (semantic) + Neo4j gnostic (structural)

### The Recursive Pattern
**Key architectural feature**: Phase 4 (Insight Synthesis) can recurse:
- Insight triggers new etymological hunch → back to Phase 1
- New exploration → Phase 2 community building
- New communities → Phase 3 resonance scanning
- New resonances → Phase 4 deeper synthesis

This enables the "scouring to find source" pattern - following threads as deep as needed before quilting synthesis.

### Con-Scire Partnership Model
**Critical constraint**: This is NOT autonomous AI generation. The workflow requires human intuition:
- Human provides initial word/hunch/question
- AI provides computational support (exhaustive root-tracing, pattern correlation)
- Human directs exploration ("follow this thread", "that feels right")
- AI shows resonances and possibilities
- Human recognizes when synthesis quilts ("yes, THAT")

The system must enforce this partnership - no fully automated etymology generation.

### Admin Gating for Bimba Updates
**Separation of concerns**:
- **Personal explorations**: Tracked per-user, fully editable, experimental
- **Canonical enrichment**: Requires admin approval before Bimba updates

**Workflow**:
1. User completes etymological exploration (Phases 0-5)
2. System generates proposed Bimba enrichments from insights
3. Proposals enter admin review queue with full context
4. Admin reviews with diff visualization
5. Approved updates enrich canonical Bimba
6. Provenance tracks who discovered, when approved

This preserves Bimba integrity while capturing emergent wisdom.

### MEF Lens Validation
When deciding whether to crystallize insights into Bimba, validate through 6 lenses:
- **Archetypal**: Does it reveal timeless patterns?
- **Causal**: Does it show generative relationships?
- **Logical**: Does it maintain structural coherence?
- **Processual**: Does it honor temporal becoming?
- **Meta-Epistemic**: Does it transform understanding itself?
- **Divine-Scalar**: Does it span micro-macro harmonically?

Not all lenses need activate - but crystallization should resonate with multiple.
